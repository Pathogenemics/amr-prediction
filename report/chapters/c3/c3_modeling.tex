\section{Rule-Based Prediction}

% Y doan: gioi thieu baseline rule-based va muc dich dua vao so sanh.
% Vai tro: dat baseline xac dinh de tach tac dong cua connection scope khoi tac dong cua hoc may.
Rule-based prediction serves as a deterministic reference baseline under the biologically constrained scopes. This direction is grounded in curated determinant resources and direct genotype-to-phenotype comparison logic, which are represented in this study by AMRFinderPlus-style determinant annotation and by genotypic-versus-phenotypic AMR comparison in \textit{Salmonella}~\citep{feldgarden2021amrfinderplus,shen2024comparison}. The baselines use the same cleaned cohort, the same normalized genotype table, and the same antibiotic-specific connection logic as the learned models, so they show how much signal is already available from retained determinant evidence before any learned weighting is introduced.

% Y doan: mo ta quy tac du doan cua strict va broad rule-based.
% Vai tro: dinh nghia ro co che baseline va ly do chi dung strict va broad.
Only the \textit{strict} and \textit{broad} scopes are used for rule-based prediction. The \textit{all} scope is excluded because it deliberately retains determinants without requiring biological relevance to the target antibiotic, so a direct deterministic match rule would no longer represent antibiotic-specific inference. In the present study, after genotype filtering and scope matching, an isolate is predicted as resistant if at least one connected determinant remains with non-zero retained evidence. Otherwise, it is predicted as susceptible. Under this design, the difference between \textit{strict rule-based} and \textit{broad rule-based} is not a different prediction formula, but the width of genotype evidence allowed to connect to the target antibiotic before the rule is applied.

% Y doan: giai thich y nghia cua baseline.
% Vai tro: noi ro vi sao can baseline nay trong thiet ke thuc nghiem.
As a result, the rule-based baselines provide a direct genotype-to-phenotype reference and help separate gains produced by broader biological scope from gains produced by machine learning itself.

\section{Machine-Learning Prediction}

% Y doan: gioi thieu phan hoc may va giai thich no dung cung du lieu da chuan bi.
% Vai tro: chuyen tu baseline sang mo hinh hoc may ma van giu nguyen bien thuc nghiem chinh.
Machine-learning prediction uses the same antibiotic-specific prepared inputs produced by the preceding data stage. In the rule-based baseline, resistance is inferred directly once at least one determinant remains connected to the target antibiotic after scope matching. In the machine-learning setting, by contrast, prediction is not triggered by a single explicit rule. Instead, multiple retained determinants can contribute jointly through their coverage, identity, and lineage-match attributes. For example, one isolate may contain several connected determinants with uneven evidence strength, and the model is allowed to learn how these signals combine rather than treating any one retained determinant as automatically sufficient. Because the biological scope is already encoded during data preparation, the learned comparison changes the way retained determinant evidence is combined rather than redefining the underlying cohort or the connection rules.

% Y doan: mo ta cau truc bang input cho mo hinh.
% Vai tro: xac dinh ro don vi mau, label, va feature representation.
\subsection{Construction of Model Inputs}

One prepared input table is constructed for each antibiotic under each connection scope. Each row represents one isolate, and the final label encoding uses \texttt{resistant = 1} and \texttt{susceptible = 0}. This encoding makes the positive class correspond directly to resistant isolates throughout training and later evaluation.

The final feature representation does not include a separate binary presence column. Instead, each retained determinant contributes three feature types: \texttt{\{element\}\_coverage}, \texttt{\{element\}\_identity}, and \texttt{\{element\}\_lineage\_match}. Presence is already implied by non-zero retained evidence, so an explicit presence flag would be redundant. This representation preserves both quantitative evidence quality and scope-related biological information while avoiding unnecessary duplication of the same determinant signal.

Prepared inputs are stored separately by scope before training begins. This keeps model fitting independent from raw TSV ingestion and hidden preprocessing inside the training workflow, so scope comparisons remain reproducible and can be traced back to fixed antibiotic-specific input tables.

% Vai tro: dinh nghia mo hinh, ly do dung one-model-per-antibiotic, va setup train nhat quan.
\subsection{Training Configuration}

The final classifier is XGBoost, and one model is trained per antibiotic. This antibiotic-specific design is necessary because resistance mechanisms, class balance, and relevant determinant sets differ across drugs. A shared global model would therefore mix prediction tasks that are biologically and statistically distinct.

The final XGBoost configuration uses \(250\) trees, learning rate \(0.05\), maximum depth \(5\), subsample \(0.8\), column subsample per tree \(0.8\), and random seed \(42\). Because resistant isolates are often the minority class, \texttt{scale\_pos\_weight} is recomputed per antibiotic from the resistant-to-susceptible imbalance in the training data. This keeps the model from being dominated by the larger susceptible class.

Training quality is estimated with five-fold stratified cross-validation. Stratification is necessary because the resistant/susceptible ratio differs substantially across antibiotics, and an unstratified split could distort the effective learning problem in smaller or more imbalanced datasets. After cross-validation metrics are collected, a final model is fitted on the full antibiotic-specific dataset so that feature-importance scores can be exported from a model trained on the complete prepared input.
